近两周总结

November 24, 2017
对知识的总结归纳有助于加深理解，并进一步提高。参照各位大牛的做法，选择总结知识的平台主要有 云服务器，Wamp或者Lamp加wordpress的形式。这种形式需要购买云主机，购买域名，并从底层开始搭建网站，需要考虑网站的安全性，点击容量，页面设计，并需要学习PHP，耗费的精力比较大，注意力没有放在写作上 纯网页版的博客，类似简书，cnblog, csdn,等这样的平台。国内很多程序员在上面开通了技术分享文章，交流氛围较为浓厚，并且专注于写作 Github pages服务，Github就不多说了，其下属的Pages服务介于服务器和纯网页Blog之间，空间容量有限，静态网页，采用Markdown形式书写，可以绑定自己的域名。并且很多大牛在上面发布代码以及开通博客。 通过比较并结合自己的兴趣情况，选择Git hub pages的形式搭建了个人技术总结平台。 经过安装Git，学习Git命令，学习Hugo，制作了一个静态网页，以后将着重于书写.md文档，也就是Markdown文档 写作计划。三维扫描重建技术方案 雕刻机设计制作 六足机器人 数控机床设计 麻醉机呼吸回路设计 梯度线圈设计 以及后续的数据分析，机器学习图像识别等领域 未来专注的领域，围绕图像视觉领域，包括三维扫描重建，主要是算法代码以及方案实现，以及硬件平台，医疗或者保健健康领域的应用，主要有内镜，肠镜，宫颈筛查等视觉应用领域。从产品，图像，算法，代码等。
